## 1. Project Overview

Custom denim manufacturing ERP for JD Denim / Sinaran (PT Triputra Textile).
Tracks full production pipeline from sales contracts through finishing.

**Tech stack:**
- Frontend: Next.js 15 + React 19 (TypeScript), Tailwind CSS v4, Framer Motion v12
- Backend: Node.js + Express (TypeScript) — `apps/api/src/routes/denim.ts`
- Database: PostgreSQL (Neon) via Prisma ORM — `packages/prisma/schema.prisma`
- Monorepo root: `~/erp-sinaran/`
- Deployed: Vercel (frontend) + Render (backend) + Neon (database)
- Git branch: `ibet/branch`

**Production pipeline (in order):**
```
Sales Contract → SACON → Warping → Indigo → Weaving → 
Inspect Gray → BBSF → Inspect Finish → Complete
```

## 2. KP Code Cipher

Format: 4 chars. Positions 1-2: A-Z alphabet. Positions 3-4: digit cipher QSDTELNJPB (Q=0...B=9)
Encoder/Decoder: `apps/api/src/lib/kp.ts`
generateNextKP() is wrapped in Prisma $transaction — race condition fixed.

## 6. SACON Pipeline Stage (NEW — added 2026-03-27)

SACON is a yarn sample sent to Jakarta HQ before warping begins.
Factory submits → Jakarta approves (ACC) or rejects (TIDAK ACC).

### API Endpoints
- POST /api/denim/sales-contracts/:kp/sacon-submit
- POST /api/denim/sales-contracts/:kp/sacon-decision
- GET  /api/denim/sacon-inbox?status=pending|awaiting_approval|all

### Pages
- /denim/inbox/sacon — Factory submits sacon (SaconInboxPage.tsx)
- /denim/approvals/sacon — Jakarta approves/rejects (SaconApprovalsPage.tsx)

### SalesContract SACON fields (already in DB)
j, j_c, b_c, tb, tb_real, bale_lusi, total_pakan, bale_pakan,
ts (tanggal sacon), sacon (boolean), acc, proses, foto_sacon
