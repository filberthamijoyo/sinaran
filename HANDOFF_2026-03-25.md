## 1. Project Overview

Custom denim manufacturing ERP for JD Denim / Sinaran (PT Triputra Textile).
Tracks full production pipeline from sales contracts through finishing.

**Tech stack:**
- Frontend: Next.js 15 + React 19 (TypeScript), Tailwind CSS v4, Framer Motion v12
- Backend: Node.js + Express (TypeScript) — `apps/api/src/routes/denim.ts`
- Database: PostgreSQL (Neon) via Prisma ORM — `packages/prisma/schema.prisma`
- Monorepo root: `~/erp-sinaran/`
- Deployed: Vercel (frontend) + Render (backend) + Neon (database)
- Git branch: `ibet/branch`

**Production pipeline (in order):**
```
Sales Contract → Warping → Indigo → Weaving → Inspect Gray → BBSF → Inspect Finish → Complete
```

## 2. KP Code Cipher

Format: 4 chars. Positions 1-2: A-Z alphabet. Positions 3-4: digit cipher QSDTELNJPB (Q=0...B=9)
Encoder/Decoder: `apps/api/src/lib/kp.ts`
generateNextKP() is wrapped in Prisma $transaction — race condition fixed.

### PHASE 2 — BBSF Missing Data

**Two new datasets to add to the database:**

**Dataset 1: PRODUKSI_BBSF** (7,332 rows)
Source: PRODUKSI_BBSF.xlsx
Structure: TGL, SHIFT, KP, KODE (construction), QTY (meters produced)
This is daily production output — how many meters came out of BBSF per KP per shift.

New Prisma model needed:
```prisma
model BBSFProductionRecord {
  id        Int       @id @default(autoincrement())
  tanggal   DateTime
  shift     String?
  kp        String
  kode      String?
  qty       Decimal?
  line      String?   // LINE 1, LINE 2, LINE 3
  createdAt DateTime  @default(now())

  @@index([kp])
  @@index([tanggal])
}
```

**Dataset 2: SUSUT_BBSF** (shrinkage measurements)
Source: SUSUT_BBSF.xlsx, use sheet "Shifted_DATA_SUSUT_BBSF"
Structure: Tanggal, No, KP_KODE (full code with construction), KERETA (cart number),
  Susut Awal (warp shrinkage start), Set Awal (%),
  Tengah (middle), Set Tengah (%),
  Akhir (end), Set Akhir (%),
  Skew Awal, Tengah Skew, Akhir Skew, Set Skew,
  + same fields repeated for Susut Pakan (weft shrinkage)

New Prisma model needed:
```prisma
model BBSFSusutRecord {
  id               Int       @id @default(autoincrement())
  tanggal          DateTime
  no               Int?
  kp               String    // extracted from KP_KODE (first 4 chars)
  kp_kode          String?   // full code e.g. "BDDP DTR DALLAS 3 P LOT P16"
  kereta           Int?      // cart number
  // Warp shrinkage (susut lusi)
  susut_lusi_awal  Decimal?
  set_lusi_awal    Decimal?
  susut_lusi_tengah Decimal?
  set_lusi_tengah  Decimal?
  susut_lusi_akhir Decimal?
  set_lusi_akhir   Decimal?
  // Weft shrinkage (susut pakan)
  susut_pakan_awal  Decimal?
  set_pakan_awal    Decimal?
  susut_pakan_tengah Decimal?
  set_pakan_tengah  Decimal?
  susut_pakan_akhir Decimal?
  set_pakan_akhir   Decimal?
  // Skew
  skew_awal        Decimal?
  skew_tengah      Decimal?
  skew_akhir       Decimal?
  set_skew         Decimal?
  keterangan       String?
  createdAt        DateTime  @default(now())

  @@index([kp])
  @@index([tanggal])
}
```
